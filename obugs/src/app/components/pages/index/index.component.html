Bug Tracking

Software developers use internal bug tracking systems daily to maintain their applications. However, their platform is usually private and not scalable for big communities. Users are most of the time in the dark, not able to know if they are adding duplicates, if the bug is already reported or if a developer is able to understand the issue.

oBugs aims to generalize bug tracking for many users to contribute.

Collaboration

When a software is used by thousands of people, wider issues get discovered over internal beta testing. However, it is hard to convey the problem back to a developer. The current tools - forums, social networks and simple forms - are inefficient to absorb user feedbacks.

With oBugs, get the bug database organized, vote to set priorities, all so that developers can focus on fixing real problems.

Free

This project is a non-profit platform. Dedicated users willing to spend some time to manage bugs improve software they use daily. Moreover, the developers are free to use the database content from their community.

No user tracking, no data selling. Simply users helping to improve software.
