Bug Tracking

Software developers use internal bug tracking systems daily to maintain their applications. However, their platform is usually private and not scalable for big communities. Finally users are most of the time in the dark, not able to know if they are adding duplicates, if the bug is already reported or if any developers is able to see the issue.

oBugs aims to generalize bug tracking for a lot of users to contribute. When social networks meet bug tracking.

Collaboration

When a software is used by thousands of people, wider issues get discovered over internal testing. However, it is hard to convey the problem back to a developer. The current tools - forums, social networks or custom tools - are inefficient to absorb user feedbacks.

With oBugs, filter the bug database, create new ones, discuss issues, vote to edit available information; all so that developers can focus on fixing real problems instead of looking around.

Free

This project is a non-profit platform. Dedicated users accept to spend some time to report bugs if they improve the piece of software they use daily. Moreover, the developers are free to use the database content their community cures for them.

No user tracking, no data selling. Simply users helping to improve software.
